# resume-hub — Design Context

## Design Context

### Users

**Primary persona**: 동시에 여러 회사에 적극 지원 중인 이직 준비자.
- 시간 압박 상태. 플랫폼마다 같은 이력을 반복 입력하는 피로에서 해방되고 싶음.
- 이직 스트레스 + "내 커리어를 잘 정돈해서 보여주고 싶다"는 욕구가 공존.
- 맥락: 점심·퇴근 후·주말에 몰아서 이력 업데이트 → 몇 분 내에 끝내고 여러 플랫폼에 자동 제출하고 싶음.
- 페이지당 예상 체류: 랜딩 ~10초, 프로필 ~3분.

**Job to be done**: "한 곳에서 내 이력을 최신·정확하게 유지하고, 플랫폼 지원서 폼은 익스텐션이 알아서 채우게 하기".

### Brand Personality

**Voice in three words**: **차분함 · 정확함 · 신뢰감있음** (Calm · Precise · Trustworthy).

**Tone**: 전문가에게 전문가로 말한다. 과장·느낌표·감탄사 없음. 직접적이되 친절. 오류 메시지도 책망하지 않음("이름을 입력해주세요" ○ / "이름은 필수입니다!" ×).

**Emotional goal**: **전문가적 자존심** — 사용자가 "이 도구를 쓰면 내 이력이 더 품질 좋아 보인다"고 느껴야 한다. 제품 자체의 타이포그래피·여백·디테일이 "잘 정돈된 문서"의 감각을 전달해 사용자의 자기 인식을 함께 끌어올린다.

### Aesthetic Direction

**Core direction**: *Editorial-grade professional tool*. 잘 조판된 단행본·프리미엄 제품 매뉴얼·고급 시계 브로셔의 감각. Notion의 warm minimalism에 에디토리얼 타이포그래피의 엄격함을 덧댄 톤.

**Theme**: 라이트(화이트 + warm white `#f6f5f4` 교차). DESIGN.md가 규정한 Notion 팔레트 전제.

**Typography posture**: NotionInter의 display letter-spacing(-2.125px @ 64px)을 실제로 살려 "문서처럼 잘 조판됨" 인상을 준다. 본문은 16px 1.50 line-height로 읽기 편하게. 폼 레이블은 14px weight 500.

**References**:
- **Notion** — warm neutrals, whisper borders, restrained motion. (이미 DESIGN.md 기반)
- **Linear (static pages)** — 정보 계층의 명료함, 과하지 않은 단일 accent.
- **Stripe docs** — 신뢰감 주는 폼 UX, 검증 오류의 조용한 처리.

**Anti-references**:
- 국내 전통 구직 포털(사람인·잡코리아·인크루트) — 촘촘한 배너, 여러 색의 경고, 테이블 중심 레이아웃.
- Y2K·레트로·플레이풀 — 네온, 글리치, 과도한 장식 금지.
- AI 미래주의 — 보라→파란 그라디언트, 글로잉 엣지, glassmorphism 금지.
- 관료적 기업 톤 — 과한 서류적 테이블, 차가운 단색 버튼, 인간미 없는 문구.

### Design Principles

1. **품질감은 세부에서 온다.** 타이포그래피의 letter-spacing·line-height, 8pt 기반의 organic 간격, whisper border의 밀도가 "이력서가 잘 정리되어 있다"는 신뢰감을 만든다. 장식적 이펙트는 품질을 대체하지 못한다.
2. **조용하지만 따뜻하게.** Warm neutral(`#f6f5f4`·`#615d59`)을 항상 중심점으로. 차가운 blue-gray도, 과장된 친근함도 거부.
3. **스트레스 받는 사용자를 거스르지 않는다.** 모달·팝업·흔들림·빨간색 경고 남용 금지. 검증 오류는 해당 필드 옆에 조용히, 구체적으로. 성공은 "저장됨" 배지 한 줄.
4. **속도를 시각적으로 증명한다.** 200ms 이하 요청엔 스피너 없이 즉시 완료 상태로 전환. 진행 중 표현은 버튼 비활성화 + 레이블 교체로 충분. 불필요한 로딩 애니메이션 금지.
5. **에디토리얼 타이포그래피를 실제로 구현한다.** 랜딩 히어로 54px/tight tracking, 프로필 페이지 섹션 라벨·본문·캡션의 위계를 5단 이상 명확히 분리. 한 페이지에 같은 크기 텍스트만 있으면 실패.

### Token Reference

모든 디자인 토큰은 `packages/ui/src/styles.css`의 `@theme` 블록에서 노출. DESIGN.md 값이 진실의 원천이며 토큰은 그 투영.

- Surface/ink: `--color-surface`, `--color-surface-warm`, `--color-ink`, `--color-ink-soft`
- Warm gray: `--color-warm-300`, `--color-warm-500`, `--color-warm-900`
- Accent: `--color-accent` (`#0075de`), `--color-accent-hover`, `--color-accent-focus`, `--color-accent-soft`
- Radius: `--radius-button` (4px), `--radius-card` (12px), `--radius-card-lg` (16px), `--radius-pill` (9999px)
- Shadows: `--shadow-card` (4-layer 누적 ≤ 0.05 opacity), border `--color-border-whisper` (1px solid rgba(0,0,0,0.1))
- Typography: `--font-sans` (NotionInter → Inter → system), `--tracking-display` (-0.033em), `--tracking-heading` (-0.023em)

### Accessibility

- **Baseline**: WCAG 2.1 AA 실질 준수 (FR-012 in specs/001-auth-profile).
- **Keyboard**: 전 UI 키보드 전주행 필수 (SC-006). 포커스 링은 `--color-accent-focus` 2px 아웃라인.
- **Reduced motion**: `prefers-reduced-motion: reduce` 시 모든 transition/animation을 `0ms` 또는 fade-only로 제한.
- **Contrast**: 텍스트-배경 ≥ 4.5:1. DESIGN.md에 이미 모든 기본 페어가 검증됨.

### Motion Budget

- Fast feedback: 100-150ms (버튼 상태 변화, 호버).
- Section/route transition: 200-300ms exponential ease-out.
- Bounce·elastic easing **금지**.
- 페이지 로드 애니메이션: 최소화. 서버 컴포넌트 스트리밍으로 "갑자기 떠 있음"에 가깝게.
